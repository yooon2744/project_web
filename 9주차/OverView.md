## react가 뭔가, css html이랑 차이점, 왜 써야하는지
- 기존 css html의 역할을 한 데 모아 자동화하듯이 써주는 역할, 컴포넌트를 써서 분할관리하는게 장점이고, 실시간 갱신도 가능

## react가 필요한 이유 
- 리액트는 데이터가 변경될 때마다 화면 요소를 조작할 필요 없이(간단함) 알아서 효율적으로 업데이트 해주고, 다수의 조작을 할때 컴포넌트를 분할해서 조립식으로 나눠서 개발 가능하기 때문에 수정관리가 용이함.


## 반복되는 UI를 컴포넌트로 분할시의 장점 
- 중복되는 작업 방지
- 유지보수가 편해짐 나누기 때문에 
- 가독성이 좋아짐
