# 2주차 - HTML 기초 1: 태그와 문서 구조

## 이번 주 목표

- HTML 태그가 `시작 태그 + 내용 + 종료 태그` 구조로 이루어진다는 점을 이해할 수 있다.
- `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`가 문서에서 어떤 역할을 하는지 설명할 수 있다.
- 태그의 중첩과 들여쓰기가 왜 중요한지 이해할 수 있다.
- `title`, `lang`, `charset`처럼 자주 보는 속성의 의미를 읽을 수 있다.

## 왜 1주차와 다르게 보나요?

`01-setup/starter/html-playground/index.html`은 "파일을 열고, 저장하고, 브라우저에서 렌더링을 확인하는 흐름"에 익숙해지는 것이 핵심이었습니다.
이번에는 같은 HTML 파일이라도 "어떤 태그가 어떤 역할을 맡는지"를 더 정확하게 읽는 연습을 합니다.

## 먼저 알아둘 개념

### 1. HTML 요소는 이렇게 생깁니다

```html
<p>안녕하세요. 저는 웹개발을 배우는 학생입니다.</p>
```

- `<p>`: 시작 태그
- `안녕하세요. 저는 웹개발을 배우는 학생입니다.`: 내용
- `</p>`: 종료 태그

### 2. 속성은 태그에 추가 정보를 붙입니다

```html
<html lang="ko">
<meta charset="UTF-8" />
```

- `lang="ko"`: 이 문서의 주 언어가 한국어라는 뜻입니다.
- `charset="UTF-8"`: 한글이 깨지지 않도록 문자 인코딩을 지정합니다.

### 3. 태그는 서로 안에 들어갈 수 있습니다

```html
<section>
  <h2>기본 정보</h2>
  <p><strong>학번:</strong> 20260001</p>
</section>
```

- `<section>` 안에 `<h2>`와 `<p>`가 들어가 있습니다.
- `<p>` 안에는 `<strong>`이 들어가 있습니다.
- 이런 구조를 읽기 쉽게 보이게 하는 것이 들여쓰기입니다.

## 코드에서 꼭 볼 부분

- `<!DOCTYPE html>`: 브라우저에 "이 문서는 HTML5 문서입니다"라고 알려줍니다.
- `<html lang="ko">`: 문서 전체를 감싸며, 언어 정보도 함께 가집니다.
- `<head>`: 브라우저 탭 제목, 문자 인코딩, 화면 크기 대응 같은 설정이 들어갑니다.
- `<title>`: 브라우저 탭에 보이는 이름입니다.
- `<body>`: 실제 화면에 보이는 콘텐츠가 들어갑니다.
- `<header>`, `<main>`, `<section>`: 콘텐츠를 의미 있는 구역으로 나눌 때 사용합니다.

## 실습 순서

1. `02-html/example/profile-page/index.html`을 열고 가장 바깥쪽 태그부터 안쪽으로 읽습니다.
2. `<head>` 안에는 화면에 직접 안 보이는 설정이, `<body>` 안에는 실제 콘텐츠가 있다는 점을 확인합니다.
3. `02-html/starter/self-intro-page/index.html`을 열고 아직 비어 있는 텍스트를 채웁니다.
4. 태그를 지우기 전에 "이 태그가 내용의 역할을 설명하는지"를 먼저 생각합니다.
5. 저장 후 브라우저에서 제목, 문단, 이미지, 표가 각각 어떻게 보이는지 확인합니다.

## 학생 실습 미션

1. `02-html/starter/self-intro-page/index.html`의 `<title>`과 `<h1>`을 자신의 정보로 수정합니다.
2. 한 줄 소개 문단을 자연스럽게 다시 작성합니다.
3. `<section>`이 몇 개 있는지 세어보고, 각 구역이 무엇을 묶는지 말로 설명합니다.
4. `<img>` 태그의 `src`와 `alt`를 읽고 각각 무엇을 담당하는지 설명합니다.
5. 표 안의 내용을 자신의 시간표나 활동 계획으로 바꿉니다.

## 체크리스트

- [ ] `head`와 `body`의 차이를 설명할 수 있다.
- [ ] 시작 태그와 종료 태그를 구분할 수 있다.
- [ ] 들여쓰기를 보고 태그의 중첩 구조를 읽을 수 있다.
- [ ] `title`, `lang`, `charset`, `alt` 같은 속성의 역할을 말할 수 있다.

## 자주 하는 실수

- 화면에 보이는 내용과 문서 설정을 같은 영역에 섞어 쓰는 경우
- 여는 태그와 닫는 태그의 짝을 놓치는 경우
- 들여쓰기를 무시해서 어떤 태그가 어디에 속하는지 헷갈리는 경우
- 속성 값을 지우고도 왜 화면이 달라졌는지 원인을 못 찾는 경우
