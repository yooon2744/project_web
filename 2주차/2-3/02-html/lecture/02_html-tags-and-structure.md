# 2주차 - HTML 기초 2: 주요 태그와 구조화

## 이번 주 목표

- 제목, 문단, 목록, 링크, 이미지, 표 태그를 구분해서 사용할 수 있다.
- 내용을 의미 있는 구역으로 나누는 구조화 방식을 이해할 수 있다.
- 같은 "보이는 모양"이라도 어떤 태그가 더 적절한지 판단할 수 있다.

## 왜 이 실습을 하나요?

HTML은 글자를 화면에 띄우는 도구가 아니라, 내용의 역할을 구분하는 도구입니다.
예를 들어 취미 목록은 그냥 줄바꿈 여러 번으로 적는 것보다 `<ul>`과 `<li>`로 표현하는 편이 더 정확합니다.
이미지도 그냥 "사진 넣기"가 아니라 `src`, `alt`, 크기 같은 정보를 함께 읽어야 합니다.

## 코드에서 꼭 볼 부분

### 제목과 문단

- `<h1>`: 페이지에서 가장 중요한 제목입니다.
- `<h2>`: 큰 주제 아래의 소제목입니다.
- `<p>`: 설명 문장, 소개글, 안내 문구를 담습니다.

### 목록

```html
<ul>
  <li>러닝</li>
  <li>영화 감상</li>
</ul>
```

- `<ul>`: 순서가 없는 목록입니다.
- `<li>`: 목록 항목 하나입니다.

### 링크

```html
<a href="https://example.com" target="_blank">자주 보는 사이트</a>
```

- `href`: 이동할 주소입니다.
- `target="_blank"`: 새 탭에서 열도록 지정합니다.

### 이미지

```html
<img
  src="../../assets/profile-placeholder.svg"
  alt="자기소개용 기본 프로필 이미지"
  width="240"
/>
```

- `src`: 어떤 이미지를 보여줄지 지정합니다.
	- `..` 은 현재 경로 기준으로 상위 폴더를 의미합니다.
		- `../../` 두 단계 상위 폴더를 의미
	- `.`은 현재 폴더를 의미합니다.
- `alt`: 이미지를 글로 설명합니다. 화면 낭독기나 이미지 로딩 실패 상황에서 중요합니다.
- `width`: 표시 크기를 조절합니다.

### 표

```html
<table border="1">
  <tr>
    <th>요일</th>
    <th>활동</th>
  </tr>
  <tr>
    <td>월요일</td>
    <td>웹 기초 복습</td>
  </tr>
</table>
```

- `<table>`: 표 전체를 감쌉니다.
- `<tr>`: 한 줄(row)입니다.
- `<th>`: 제목 칸입니다.
- `<td>`: 내용 칸입니다.

## 구조를 읽는 기준

- 소개 문장이라면 `<p>`가 자연스럽습니다.
- 여러 항목을 나열한다면 `<ul>`과 `<li>`가 자연스럽습니다.
- 외부 사이트로 이동한다면 `<a>`가 자연스럽습니다.
- 시간표나 비교표처럼 "행과 열"이 있다면 `<table>`이 자연스럽습니다.
- 서로 다른 주제를 구역으로 나눈다면 `<section>`이 자연스럽습니다.

## 실습 순서

1. `02-html/example/profile-page/index.html`에서 `<section>`이 어디서 시작하고 끝나는지 먼저 찾습니다.
2. 목록, 링크, 이미지, 표 태그를 각각 찾아서 어떤 데이터를 표현하는지 확인합니다.
3. `02-html/starter/self-intro-page/index.html`에서 취미 목록을 자신의 정보로 바꿉니다.
4. 링크 문구와 `href`를 자신이 자주 보는 사이트로 수정합니다.
5. 표의 행과 열이 어떤 기준으로 나뉘는지 읽고, 자신의 일정으로 다시 채웁니다.

## 학생 실습 미션

1. `02-html/starter/self-intro-page/index.html`의 취미를 3개 이상 작성합니다.
2. `02-html/starter/self-intro-page/index.html`의 링크를 실제 자주 방문하는 사이트로 수정합니다.
3. `02-html/starter/self-intro-page/index.html`의 표를 "요일 / 활동 / 메모" 기준으로 다시 작성합니다.
4. 이미지를 원하는 것으로 바꾸고(png 파일), `alt` 문구를 더 구체적으로 고쳐봅니다.
5. 왜 취미는 `<table>`보다 `<ul>`이 더 자연스러운지 설명해봅니다.

## 체크리스트

- [ ] 목록, 링크, 이미지, 표 태그를 각각 찾고 설명할 수 있다.
- [ ] `href`, `src`, `alt`, `width` 같은 속성을 읽을 수 있다.
- [ ] 내용을 주제별 섹션으로 나눌 수 있다.
- [ ] "모양"이 아니라 "의미" 기준으로 태그를 고를 수 있다.

## 자주 하는 실수

- 목록인데도 `<p>`만 여러 개 쓰는 경우
- `href`나 `src`를 비워두고 링크나 이미지가 왜 동작하지 않는지 헷갈리는 경우
- 이미지 설명 없이 `alt=""`를 남겨두는 경우
- 표가 아닌 데이터에도 무조건 `<table>`을 쓰는 경우

## 한 줄 정리

좋은 HTML은 태그를 많이 쓰는 것이 아니라, 내용에 맞는 태그를 선택해서 구조를 또렷하게 만드는 것입니다.
