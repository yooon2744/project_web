# 2주차 - HTML 기초 3: 폼과 입력 태그

## 이번 주 목표

- `form`, `label`, `input`, `textarea`, `select`, `button`의 역할을 이해할 수 있다.
- 입력 요소마다 어떤 데이터를 받는지 구분할 수 있다.
- 폼 안에서도 의미 있는 구조를 만들기 위해 `fieldset`과 `legend`를 읽을 수 있다.

## 왜 이 실습을 하나요?

앞선 파일들이 "읽는 화면"이었다면, 폼은 "사용자가 입력하는 화면"입니다.
앞으로 JavaScript에서 이벤트를 다루고 서버로 데이터를 보내려면, 먼저 HTML 폼이 어떤 입력칸들로 구성되는지 알아야 합니다.

## 이번 주 파일 설명

- `02-html/example/campus-form/index.html`: 동아리 신청서 형태로 여러 입력 태그를 모아 둔 예제입니다.

## 코드에서 꼭 볼 부분

### 폼의 바깥 구조

- `<form>`: 여러 입력 요소를 하나의 제출 단위로 묶습니다.
- `<fieldset>`: 관련 있는 입력 묶음을 그룹으로 나눕니다.
- `<legend>`: 그 그룹의 제목입니다.

### 입력 태그

- `<label for="student-name">`: 어떤 입력칸인지 설명합니다.
- `<input type="text">`: 한 줄 텍스트를 입력받습니다.
- `<input type="email">`: 이메일 형식 입력에 자주 사용합니다.
- `<textarea>`: 여러 줄 텍스트를 입력받습니다.
- `<select>`: 정해진 선택지 중 하나를 고르게 합니다.
- `<button type="submit">`: 폼을 제출하는 버튼입니다.

## 꼭 같이 봐야 하는 연결

```html
<label for="student-name">이름</label>
<input id="student-name" type="text" />
```

- `for="student-name"`와 `id="student-name"`가 연결되어 있습니다.
- 이렇게 연결해 두면 label을 눌렀을 때 해당 입력칸에 초점이 이동합니다.

## 실습 순서

1. `02-html/example/campus-form/index.html`을 열고 `<form>` 안에 어떤 입력 요소가 있는지 먼저 세어봅니다.
2. 각 `<label>`이 어떤 `id`와 연결되는지 확인합니다.
3. `text`, `email`, `select`, `textarea`가 각각 어떤 입력 상황에 어울리는지 구분합니다.
4. placeholder 문구와 버튼 문구를 바꿔 보며 폼이 어떤 목적의 화면인지 더 분명하게 만듭니다.

## 학생 실습 미션

1. 연락처를 받는 입력칸을 하나 더 추가해봅니다.
2. 지원 분야를 고르는 `<select>` 항목을 3개 이상으로 바꿔봅니다.
3. 자기소개 입력칸의 안내 문구를 더 구체적으로 수정합니다.
4. 버튼 문구를 `지원서 제출하기`로 바꿉니다.
5. 어떤 항목에 `<input>`이 맞고, 어떤 항목에 `<textarea>`가 맞는지 설명합니다.

## 체크리스트

- [ ] `form`과 `label`의 역할을 설명할 수 있다.
- [ ] `for`와 `id`가 왜 함께 쓰이는지 이해했다.
- [ ] 입력 목적에 따라 적절한 입력 태그를 고를 수 있다.
- [ ] 폼 구조를 읽고 직접 문구를 수정했다.

## 자주 하는 실수

- `label` 없이 입력칸만 만드는 경우
- 여러 줄 설명이 필요한데도 무조건 `<input>`만 쓰는 경우
- 선택지가 정해져 있는데도 텍스트 입력칸으로 받는 경우
- 버튼을 눌렀다고 해서 자동으로 저장까지 된다고 오해하는 경우

## 한 줄 정리

폼은 "사용자가 정보를 입력하는 구조"이고, 좋은 폼 HTML은 입력칸의 목적을 태그만 봐도 알 수 있게 만드는 것입니다.
