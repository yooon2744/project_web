# 3주차 - CSS 기초 3: Flexbox와 반응형

## 이번 주 목표

- `display: flex`의 역할을 이해할 수 있다.
- 가로, 세로 정렬 속성을 사용할 수 있다.
- 화면 폭이 달라질 때 배치가 어떻게 변하는지 관찰할 수 있다.

## 왜 이 실습을 하나요?

실제 웹페이지는 요소 하나만 꾸미는 것이 아니라 여러 요소를 함께 배치해야 합니다.
Flexbox를 익히면 카드 정렬, 버튼 배치, 화면 중앙 정렬 같은 작업을 훨씬 쉽게 처리할 수 있습니다.

## 준비물

- VS Code
- 브라우저
- `03-css/example/campus-landing/index.html`
- `03-css/example/campus-landing/style.css`

## 이번 주 파일 설명

- `03-css/example/campus-landing/index.html`: 랜딩 페이지 구조입니다.
- `03-css/example/campus-landing/style.css`: 카드와 버튼을 정렬하는 Flexbox 코드가 들어 있습니다.

## 코드에서 꼭 볼 부분

- `display: flex`: Flexbox 레이아웃을 시작합니다.
- `justify-content`: 주축 방향 정렬을 제어합니다.
- `align-items`: 반대축 정렬을 제어합니다.
- `gap`: 요소 사이 간격을 만듭니다.

## 실습 순서

1. 랜딩 페이지의 카드 영역을 확인합니다.
2. 카드들이 어떻게 가로로 배치되는지 찾습니다.
3. `gap` 값을 바꿔 카드 사이 간격을 조절합니다.
4. 버튼 정렬과 카드 정렬이 각각 어떤 속성으로 제어되는지 읽어봅니다.

## 학생 실습 미션

1. 카드 간격을 변경해보세요.
2. 히어로 버튼 모서리를 더 둥글게 바꿔보세요.
3. 카드 영역을 세로 배치로 바꾸면 어떻게 보이는지 실험해보세요.

## 체크리스트

- [ ] `display: flex`를 찾을 수 있다.
- [ ] `justify-content`와 `align-items`의 역할을 구분할 수 있다.
- [ ] 배치 값을 수정해 결과를 확인했다.

## 자주 하는 실수

- `display: flex` 없이 정렬 속성만 쓰는 경우
- 어느 요소가 부모 컨테이너인지 놓치는 경우
- 정렬 방향을 생각하지 않고 속성만 외우는 경우

## 한 줄 정리

Flexbox는 여러 요소를 "줄 세우고 정렬하는 도구"라고 이해하면 시작이 쉽습니다.
