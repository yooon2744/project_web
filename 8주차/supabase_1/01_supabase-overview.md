# 8주차 - Supabase : 과제내용 

## Supabase가 무엇인가?
- 웹 서비스 제작 및 활용에 필요한 db, 로그인 등의 기능을 서버구축 없이 제공하는 백엔드 역할을 해주는 클라우드 서비스


## 브라우저 화면과 데이터 저장소의 역할 차이
- 브라우저 : 이용자에게 DB 제공 및 입력
- 데이터 저장소 : 입력된 DB 보관 및 저장, 기억


## Supabase가 없는 경우의 불편함
- 새로고침이나 창을 닫거나 하면 데이터가 사라질 수 있고(저장의 문제), 다른 사용자와의 공유가 힘들다.


## id가 필요한 이유
- 데이터 분류, 식별용으로 필요함


## is_done에 들어갈 값
- 참(true) , 거짓(false)만 가지는 논리형 데이터(bool) 값이 들어감.


## created_at이 없는 경우의 불편함
- 데이터 생성 날짜 및 정렬의 문제가 생길 수 있음, 기록의 문제성


## 조회 코드와 추가 코드의 차이점
- 조회(.select()) : DB를 저장소에서 가져와서 보여줌
- 추가(.insert()) : DB를 저장소에서 넣음
